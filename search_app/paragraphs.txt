Plan selection aims to select the optimal query plan, usually the minimum latency plan. This is achieved using rule-based or cost-based selection. (1) Rule Based. For VDBMSs that only support a small number of plans, simple rules can be sufficient for plan selection. For example, AnalyticDB-V [84] and Milvus [6, 79] devise costs for several vector operators in order to use a linear cost model that aggregates the I/O and computation cost of each plan operator in order to yield a total plan cost.
